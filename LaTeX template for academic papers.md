[GitHub link](https://github.com/pmichaillat/latex-paper/tree/main) က Paul Michaillat ရဲ့ **"LaTeX template for academic papers"** ဖြစ်ပါတယ်။ ဒီ template က ရိုးရိုးရှင်းရှင်းနဲ့ အရမ်း Professional ကျလို့ နာမည်ကြီးပါတယ်။ 

---

### ၁။ File တစ်ခုချင်းစီရဲ့ တာဝန်များ (Project Structure)

ဒီ project ကို Overleaf မှာ တင်လိုက်ရင် အဓိက ဒီဖိုင်တွေကို တွေ့ရပါလိမ့်မယ် -

* **`paper.tex` (The Main File):**
    ဒါက အစ်ကို အဓိက ကိုင်ရမယ့်ဖိုင်ပါ။ စာတမ်းရဲ့ ခေါင်းစဉ်၊ နာမည်နဲ့ စာသား (Content) တွေကို ဒီထဲမှာပဲ ရေးရပါတယ်။
* **`paper.sty` (The Designer):**
    ဒါက အရေးကြီးဆုံးဖိုင်ပါ။ LaTeX ရဲ့ margin တွေ၊ font တွေ၊ `\maketitle` ရဲ့ ပုံစံတွေကို ဒီထဲမှာ ကြိုတင် Design ဆွဲထားတာပါ။ အစ်ကိုက `\usepackage{paper}` လို့ ခေါ်လိုက်တာနဲ့ ဒီဖိုင်ထဲက design တွေ အကုန်လုံး အသက်ဝင်လာတာ ဖြစ်ပါတယ်။
* **`paper.bib` (References):**
    ဒါက အကိုကိုးကားမယ့် စာအုပ်တွေ၊ စာတမ်းတွေရဲ့ စာရင်းကို သိမ်းတဲ့နေရာပါ။
* **`figures.pdf`:**
    ဒါကတော့ စာတမ်းထဲမှာ သုံးမယ့် ပုံတွေကို စုထားတဲ့ ဖိုင်ပါ။

### ၂။ ဒီ Template ရဲ့ ထူးခြားချက် (The Method)

ဒီ template က **"Separation of Concerns"** ဆိုတဲ့ programming concept ကို သုံးထားပါတယ်။

1.  **Metadata အရင်ပေးရတယ်:** `\title`, `\author`, `\date` စတာတွေကို အရင်သတ်မှတ်ရတယ်။
2.  **Custom Commands:** သူက `\paperurl` လိုမျိုး ကိုယ်ပိုင် command အသစ်တွေ ဖန်တီးထားပါတယ်။ ဒါကြောင့် standard LaTeX ထက်စာရင် ပိုပြီး သပ်ရပ်တာပါ။
3.  **Titlepage Environment:** ခေါင်းစဉ်နဲ့ Abstract ကို `\begin{titlepage} ... \end{titlepage}` ကြားထဲမှာ ထည့်ထားတဲ့အတွက် Compile လုပ်လိုက်ရင် အဲဒီအပိုင်းက စာမျက်နှာ အစမှာပဲ သီးသန့် သပ်သပ်ရပ်ရပ် ပေါ်လာတာ ဖြစ်ပါတယ်။

---

### ၃။ ဘာကြောင့် အစဉ်လိုက် ဖြစ်နေတာလဲ? (Detailed Logic)

အစ်ကို့ code ထဲမှာ `\section{Introduction}` ကို `\maketitle` ရဲ့ အပေါ်မှာ သွားရေးထားရင်တောင် `paper.sty` ထဲက rule တွေကြောင့် LaTeX က ခေါင်းစဉ်ကို အရင်ထုတ်ပေးမှာပါ။

**အလုပ်လုပ်ပုံ အဆင့်ဆင့်:**
1.  **Reading `paper.sty`:** LaTeX က အစ်ကိုသုံးမယ့် design rules တွေကို အရင်ဖတ်ပါတယ်။
2.  **Collecting Data:** `\title` နဲ့ `\author` ထဲက စာသားတွေကို မှတ်သားထားပါတယ်။
3.  **Executing `\maketitle`:** ဒီ command ကို တွေ့တာနဲ့ သူမှတ်ထားတဲ့ Title ကို ပုံစံချပြီး စာမျက်နှာထိပ်မှာ တင်လိုက်ပါတယ်။
4.  **Flowing Content:** အဲဒီနောက်မှာမှ `\section` တွေ၊ `\paragraph` တွေကို တစ်ခုပြီးတစ်ခု စီပေးသွားတာပါ။

LaTeX ရဲ့ ထူးခြားချက်က Code ထဲမှာ ရေးချင်သလို ရေးထားပေမဲ့ Compile လုပ်လိုက်တဲ့အခါ သူသတ်မှတ်ထားတဲ့ ပုံစံအတိုင်း အစဉ်လိုက် ထွက်လာတာပါ။ ဒါကို "Logical Structuring" လို့ ခေါ်ပါတယ်။

---

### ၄။ အစ်ကို့အတွက် အကြံပြုချက်
ဒီ template ကို သုံးမယ်ဆိုရင် -
* `paper.sty` ဖိုင်ကို **သွားမပြင်ပါနဲ့** (အကယ်၍ design ကို အရမ်းကျွမ်းကျင်တာမဟုတ်ရင်)။
* စာသားတွေကို `paper.tex` ထဲမှာပဲ အာရုံစိုက်ရေးပါ။
* အကို့ရဲ့ references တွေကို `paper.bib` ထဲမှာ သွားထည့်ပါ။

---
LaTeX မှာ "Separation of Concerns" ဆိုတာကတော့ **အသွင်အပြင် (Design)**၊ **အချက်အလက် (Content)** နဲ့ **ကိုးကားချက် (References)** တွေကို တစ်နေရာတည်းမှာ ရောမရေးဘဲ ဖိုင်ခွဲပြီး စီမံတာကို ဆိုလိုတာပါ။

အစ်ကိုသုံးနေတဲ့ template ထဲက ဖိုင်တစ်ခုချင်းစီ ဘယ်လိုချိတ်ဆက်ပြီး အလုပ်လုပ်လဲဆိုတာကို အဆင့်ဆင့် အသေးစိတ် ရှင်းပြပေးပါမယ်။

---

## ၁။ `paper.sty` (The Rulebook/Stylist)
ဒါကတော့ **"ဒီဇိုင်းဆရာ"** ဖြစ်ပါတယ်။ ပရိုဂရမ်မင်းမှာဆိုရင် CSS ဖိုင်နဲ့ တူပါတယ်။

* **သူ့ရဲ့တာဝန်:** စာမျက်နှာ margin ဘယ်လောက်ထားမလဲ၊ Font ဘာသုံးမလဲ၊ `\maketitle` နှိပ်ရင် ခေါင်းစဉ်ကို ဘယ်နေရာမှာ ဘယ်လိုအရွယ်အစားနဲ့ ပြမလဲဆိုတဲ့ "ညွှန်ကြားချက်" တွေ အကုန်လုံး ဒီထဲမှာ ရှိပါတယ်။
* **ဘယ်လိုအလုပ်လုပ်လဲ:** အစ်ကိုက `paper.tex` ထဲမှာ `\usepackage{paper}` လို့ ရေးလိုက်တာနဲ့ LaTeX က ဒီ `paper.sty` ကို အရင်ဆုံး စဖတ်ပါတယ်။ အဲဒီအခါမှာ `\paperurl` လိုမျိုး Command အသစ်တွေကို သူက စက်ထဲမှာ မှတ်သားလိုက်ပါတယ်။

---

## ၂။ `paper.tex` (The Content Provider)
ဒါကတော့ အစ်ကို အဓိက အလုပ်လုပ်ရမယ့် **"အချက်အလက်စင်တာ"** ဖြစ်ပါတယ်။

* **သူ့ရဲ့တာဝန်:** စာတမ်းရဲ့ ခေါင်းစဉ်၊ နာမည်၊ ပြီးတော့ တကယ့်စာသား (Body text) တွေကို ဒီထဲမှာပဲ ရေးရပါတယ်။
* **အဆင့်ဆင့် အလုပ်လုပ်ပုံ:**
    1.  **Metadata Definition:** အစ်ကိုက `\title{...}` နဲ့ `\author{...}` လို့ ရေးတဲ့အခါ LaTeX က ဒါတွေကို စာမျက်နှာပေါ် တန်းမတင်သေးဘဲ Memory ထဲမှာ ခဏသိမ်းထားပါတယ်။
    2.  **Execution (`\maketitle`):** စာသားတွေ စတော့မယ့် `\begin{document}` အောက်မှာ `\maketitle` ကို ခေါ်လိုက်တဲ့အခါမှသာ Memory ထဲက data တွေကို ယူပြီး `paper.sty` ထဲက design အတိုင်း စာမျက်နှာထိပ်မှာ ပုံဖော်လိုက်တာပါ။

---

## ၃။ `paper.bib` (The Library/Database)
ဒါကတော့ အစ်ကိုကိုးကားသမျှ စာအုပ်စာရင်းတွေ သိမ်းတဲ့ **"ဒေတာဘေ့စ်"** ဖြစ်ပါတယ်။

* **သူ့ရဲ့တာဝန်:** စာတမ်းထဲမှာ ကိုးကားထားတဲ့ သုတေသနစာတမ်းတွေရဲ့ အချက်အလက် (စာရေးသူ၊ ခုနှစ်၊ ခေါင်းစဉ်) တွေကို BibTeX format နဲ့ သိမ်းထားတာပါ။
* **ဘယ်လိုအလုပ်လုပ်လဲ:** စာသားထဲမှာ `\cite{author_key}` လို့ သုံးလိုက်ရင် LaTeX က ဒီ `paper.bib` ထဲမှာ အဲဒီ key ကို သွားရှာပါတယ်။ တွေ့ပြီဆိုရင် စာတမ်းရဲ့ အောက်ဆုံးက References section မှာ အလိုအလျောက် စာရင်းသွင်းပေးပါတယ်။

---

## ၄။ Compilation Workflow (အလုပ်လုပ်ပုံ အဆင့်ဆင့်)

အစ်ကို **Recompile** နှိပ်လိုက်တဲ့အခါ ဖြစ်သွားတဲ့ ဖြစ်စဉ်က ဒီလိုပါ-

1.  **Load Styling:** `paper.tex` ကနေ `paper.sty` ကို လှမ်းခေါ်တယ်။ Design rules တွေကို Load လုပ်တယ်။
2.  **Store Metadata:** `\title` နဲ့ `\author` ထဲက data တွေကို သိမ်းတယ်။
3.  **Construct Title:** `\maketitle` ကို တွေ့တာနဲ့ အပေါ်ဆုံးမှာ ခေါင်းစဉ်အပိုင်းကို တည်ဆောက်တယ်။
4.  **Process Text:** အောက်က `\section`, `\subsection` စတာတွေကို sequential (အစဉ်လိုက်) အလုပ်လုပ်ပြီး စာမျက်နှာတွေပေါ် စီချတယ်။
5.  **Fetch References:** `\cite` တွေ့ရင် `paper.bib` ကနေ data ဆွဲယူပြီး စာမျက်နှာအောက်ခြေနဲ့ References မှာ နံပါတ်စဉ်တပ်ပေးတယ်။

---

### အနှစ်ချုပ်
* **`paper.sty`** = Design (ဘယ်လိုပုံစံ ဖြစ်ရမလဲ)
* **`paper.tex`** = Body (ဘာတွေ ရေးမှာလဲ)
* **`paper.bib`** = Sources (ဘယ်သူ့ဆီက ယူတာလဲ)

ဒီလို ခွဲရေးထားတဲ့အတွက် အစ်ကိုက စာသားတွေ အများကြီး ပြင်နေပေမဲ့ Design ကတော့ `paper.sty` ထဲကအတိုင်း အမြဲတမ်း တသတ်မှတ်တည်း လှပနေမှာ ဖြစ်ပါတယ်။

---

ဒီ `paper.sty` ဖိုင်ဟာ အစ်ကို့ရဲ့ စာတမ်းတစ်ခုလုံးကို Professional အကျဆုံးဖြစ်အောင် ဖန်တီးပေးထားတဲ့ **"Master Design Blueprint"** ဖြစ်ပါတယ်။ Programming အရပြောရရင် ဒါဟာ UI/UX Library တစ်ခုကို ကိုယ်တိုင် ရေးထားတာနဲ့ တူပါတယ်။

---

## ၁။ Fonts & Encoding (စာလုံးအမျိုးအစားများ)
ဒီအပိုင်းက စာတမ်းရဲ့ ပထမဆုံးအမြင်ကို ဆုံးဖြတ်တာပါ။
* **Source Serif Pro:** ဒါကို main font အနေနဲ့ သုံးထားပါတယ်။ မျက်စိအေးပြီး ဖတ်ရတာ သက်သာစေတဲ့ font မျိုးပါ။
* **Source Code Pro:** Coding ရိုက်တဲ့အခါ သုံးတဲ့ font ပါ။
* **Glyphtounicode:** PDF ကို ရှာဖွေရလွယ်အောင် (Searchable) နဲ့ Copy ကူးလို့ရအောင် လုပ်ပေးပါတယ်။

## ၂။ Math Fonts (သင်္ချာပုံသေနည်းများ)
ဒီ template ရဲ့ အထူးခြားဆုံးအပိုင်းပါ။
* **Euler Greek & Fourier:** သင်္ချာသင်္ကေတတွေကို standard LaTeX ထက်စာရင် ပိုပြီး လှပတဲ့ Design တွေနဲ့ အစားထိုးပေးထားပါတယ်။
* **MTsetmathskips:** `f, y, p, j` စတဲ့ စာလုံးတွေ သင်္ချာပုံသေနည်းထဲမှာ ပါလာရင် တစ်လုံးနဲ့တစ်လုံး မထပ်အောင် Space (အကွာအဝေး) ကို Manual ညှိပေးထားတာပါ။ (ဒါဟာ အဆင့်မြင့် Design စနစ် ဖြစ်ပါတယ်)။

## ၃။ Page Layout & Spacing (စာမျက်နှာ အထားအသို)
* **Geometry:** Margin တွေကို စံသတ်မှတ်ချက်အတိုင်း (ဘယ်၊ ညာ ၁.၃ လက်မ) ညှိပေးထားပါတယ်။
* **Microtype:** စာကြောင်းတစ်ကြောင်းရဲ့ အဆုံးမှာ စာလုံးတွေ ပြတ်မသွားအောင်နဲ့ စာသားတွေကြားက space တွေကို မျက်စိနဲ့ကြည့်ရင် ညီညာနေအောင် ညှိပေးတဲ့ အဆင့်မြင့် package ဖြစ်ပါတယ်။
* **Frenchspacing:** စာကြောင်းတစ်ကြောင်းအဆုံး (Full stop) ပြီးရင် space အပိုတွေ မကျန်အောင် လုပ်ပေးပါတယ်။

## ၄။ Custom Headers & Footers (ခေါင်းစဉ်နှင့် အောက်ခြေ)
* **Fancyhdr:** စာမျက်နှာ နံပါတ်တွေကို အလယ်မှာ ထားပါတယ်။
* **\paperurl:** ဒါက အစ်ကို့အတွက် ဖန်တီးပေးထားတဲ့ **Custom Command** ပါ။ Title page ရဲ့ အောက်ခြေမှာ "Published in..." နဲ့ Website link ကို မီးခိုးရောင် စာသားလေးတွေနဲ့ လှလှပပ ဖော်ပြပေးပါတယ်။

## ၅။ Formatting Sections (ခေါင်းစဉ်ပုံစံများ)
* **Titlesec:** Standard LaTeX မှာ ခေါင်းစဉ်တွေက အရမ်းကြီးပါတယ်။ ဒီထဲမှာတော့:
    * **Section:** စာမျက်နှာရဲ့ အလယ် (Center) မှာ စာလုံးအထူနဲ့ လှလှပပ ပြပေးပါတယ်။
    * **Paragraph:** စာကြောင်းအတွင်းမှာပဲ ခေါင်းစဉ်ကို အစောင်း (Italic) ပုံစံနဲ့ ပြပေးပါတယ်။

## ၆။ Figures & Tables (ပုံနှင့် ဇယားများ)
ဒီနေရာမှာ Professional အကျဆုံး rule တွေ ပါဝင်ပါတယ်:
* **Booktabs:** ဇယားတွေမှာ ဒေါင်လိုက်မျဉ်းတွေ မသုံးဘဲ အလျားလိုက်မျဉ်း အထူ/အပါးနဲ့တင် Professional ဆန်အောင် လုပ်ပေးပါတယ်။
* **Small Caps:** Table နဲ့ Figure ဆိုတဲ့ စာလုံးတွေကို စာလုံးအကြီးအသေး (Small Caps) နဲ့ ပြပေးပါတယ်။
* **\note:** ဒါကလည်း command အသစ်ပါ။ ဇယားရဲ့ အောက်မှာ ရှင်းလင်းချက် (Note) ရေးချင်ရင် ဒီ command တစ်ခုတည်းနဲ့တင် အလိုအလျောက် spacing တွေ ညှိပေးမှာ ဖြစ်ပါတယ်။

## ၇။ Theorems & Proofs (သီအိုရီနှင့် သက်သေပြချက်များ)
ဒီ template က စီးပွားရေးနဲ့ သင်္ချာစာတမ်းတွေအတွက် ရည်ရွယ်တဲ့အတွက် `Theorem`, `Lemma`, `Proposition` စတာတွေကို နံပါတ်စဉ်တပ်ပြီး ဖော်ပြဖို့ Style တွေ ကြိုဆောက်ထားပါတယ်။ အစ်ကိုက `\begin{theorem} ... \end{theorem}` လို့ ရေးလိုက်ရုံပါပဲ။

## ၈။ Appendix (နောက်ဆက်တွဲ)
ဒီအပိုင်းက `thebibliography` (References) ပြီးတဲ့အခါ အလိုအလျောက် အသက်ဝင်လာတာပါ။
* သူက နံပါတ်စဉ်တွေကို `1, 2, 3` အစား `A1, A2, A3` ဆိုပြီး ပြောင်းပေးပါတယ်။
* ဒါကြောင့် နောက်ဆက်တွဲမှာ ပုံတွေ၊ ဇယားတွေ ထပ်ထည့်ရင် နံပါတ်တွေ မရောတော့ဘဲ စနစ်တကျ ရှိနေမှာပါ။

### အစ်ကို့အတွက် အကြံပြုချက် (Tip)
ဒီ `paper.sty` ထဲက ကုဒ်တွေကို အစ်ကိုက သွားပြင်စရာ မလိုပါဘူး။ ဒါပေမဲ့ အဲဒီထဲမှာ သုံးထားတဲ့ **Custom Commands** တွေကိုတော့ သိထားရင် ပိုကောင်းပါတယ်။ ဥပမာ -
1.  **`\paperurl{link}`**: Title page မှာ link ထည့်ဖို့။
2.  **`\note{text}`**: Table/Figure အောက်မှာ ရှင်းလင်းချက်ရေးဖို့။
3.  **`\begin{theorem}`**: သီအိုရီတွေ ရေးဖို့။

ဒီ template က အစ်ကို့ကို Software Engineer တစ်ယောက်အနေနဲ့ စာတမ်းတွေ ရေးတဲ့အခါ Design အပိုင်းကို လုံးဝ ခေါင်းစားစရာမလိုအောင် လုပ်ပေးထားတာ ဖြစ်ပါတယ်ဗျ။

---

ဒီ Template ရဲ့ Design ကို ပြင်ချင်ရင် နည်းလမ်း (၂) သွယ် ရှိပါတယ်။ တစ်ခုက `paper.sty` ထဲမှာ အမြဲတမ်း (Global) ပြောင်းလဲသွားအောင် ပြင်တာဖြစ်ပြီး၊ နောက်တစ်ခုကတော့ `paper.tex` ထဲမှာပဲ သီးသန့် (Local) ပြင်တာပါ။

## ၁။ စာလုံးအရွယ်အစားနှင့် စာရွက်အမျိုးအစား ပြောင်းလဲခြင်း
ဒါကိုတော့ `paper.tex` ရဲ့ ပထမဆုံးစာကြောင်းမှာတင် ပြင်လို့ရပါတယ်။
* **Standard:** `\documentclass[11pt, letterpaper, leqno]{article}`
* **ပြောင်းလဲလိုလျှင်:** `11pt` ကို `12pt` ပြောင်းတာမျိုး၊ `letterpaper` ကို `a4paper` ပြောင်းတာမျိုး လုပ်နိုင်ပါတယ်။

## ၂။ Margin (ဘေးအကွာအဝေး) ပြင်ခြင်း
`paper.sty` ထဲက `geometry` package နေရာမှာ သွားပြင်ရပါမယ်။
```latex
% paper.sty ထဲက ဒီစာကြောင်းကို ရှာပါ
\usepackage[top=1in, left=1.3in, right=1.3in, bottom=1.2in, footskip=0.7in]{geometry}
```
* အကယ်၍ စာမျက်နှာ ဘေးတွေကို ပိုကျဉ်းချင်ရင် `left=1in, right=1in` လို့ ပြောင်းနိုင်ပါတယ်။

## ၃။ ခေါင်းစဉ် (Sections) အရောင်နှင့် ပုံစံပြောင်းခြင်း
အကယ်၍ Section ခေါင်းစဉ်တွေကို အပြာရောင် ပြောင်းချင်တာမျိုး သို့မဟုတ် ဘယ်ဘက်ကပ်ချင်တာမျိုးဆိုရင် `paper.sty` ထဲက `titlesec` အပိုင်းမှာ ပြင်ရပါတယ်။

* **အရောင်ပြောင်းရန်:**
```latex
% paper.sty ထဲက \titleformat ကို ရှာပြီး \color{blue} ထည့်ပါ
\titleformat*{\section}{\centering\large\bfseries\color{blue}}
```

## ၄။ Line Spacing (စာကြောင်းအကွာအဝေး) ပြင်ခြင်း
ဒီ Template က standard အားဖြင့် ၁.၅ (One-half) ထားပေးထားပါတယ်။
* **Double Space ပြောင်းရန်:** `paper.sty` ထဲက `\usepackage[onehalfspacing]{setspace}` ကို `\usepackage[doublespacing]{setspace}` လို့ ပြောင်းပါ။
* **Single Space ပြောင်းရန်:** `\usepackage{setspace}` လို့ပဲ ရေးပြီး `\singlespacing` ကို သုံးပါ။

---

## ၅။ Hyperlink အရောင်များ သတ်မှတ်ခြင်း
အခု template မှာ link တွေကို `hidelinks` (အရောင်မပါ) လုပ်ထားပါတယ်။ ဒါကို အရောင်ပြောင်းချင်ရင် `paper.sty` ရဲ့ အောက်ဆုံးစာကြောင်းမှာရှိတဲ့ `hyperref` နေရာမှာ ပြင်ရပါမယ်။

```latex
% paper.sty အောက်ဆုံးမှာ ဒါကို အစားထိုးပါ
\usepackage[colorlinks=true, linkcolor=blue, urlcolor=cyan]{hyperref}
```

## ၆။ Tables နှင့် Figures ၏ Font Size ပြင်ခြင်း
ဇယားတွေထဲက စာလုံးတွေက သေးလွန်းတယ်လို့ ခံစားရရင် `paper.sty` ထဲက ဒီနေရာမှာ ပြင်နိုင်ပါတယ် -
```latex
% ဇယားထဲက စာလုံးအရွယ်အစားကို \footnotesize (အသေး) မှ \small (အလတ်) ပြောင်းခြင်း
\BeforeBeginEnvironment{tabular*}{\small}
```

### ⚠️ သတိထားရန် (Best Practice)
1.  **Original ကို သိမ်းထားပါ:** `paper.sty` ကို မပြင်ခင် မူရင်း ကုဒ်တွေကို Comment (`%`) ခံပြီးမှ အောက်မှာ အသစ်ထပ်ရေးပါ။ ဒါမှ အဆင်မပြေရင် မူလအတိုင်း ပြန်ပြင်လို့ ရမှာပါ။
2.  **Package Conflict:** Package အသစ်တွေ ထပ်ထည့်ချင်ရင် `paper.sty` ရဲ့ အပေါ်ဆုံးနားမှာပဲ စုထည့်ပါ။
3.  **Recompile:** ပြင်ပြီးတိုင်း အပြောင်းအလဲကို သိနိုင်ဖို့ `Recompile` နှိပ်ဖို့ မမေ့ပါနဲ့။

---

`paper.bib` ဖိုင်ထဲမှာ အချက်အလက်တွေ ထည့်သွင်းတာဟာ Programming မှာ Database တစ်ခုထဲကို Data ထည့်တာနဲ့ တူပါတယ်။ တစ်ခါထည့်ထားရုံနဲ့ စာတမ်းရဲ့ ဘယ်နေရာမှာမဆို လှမ်းခေါ်သုံး (Reference ပေး) လို့ ရပါတယ်။

အဆင့်ဆင့် လုပ်ဆောင်ပုံကို ရှင်းပြပေးပါမယ်။

---

## ၁။ BibTeX Entry တစ်ခုရဲ့ တည်ဆောက်ပုံ
`paper.bib` ထဲမှာ Data ထည့်ရင် အောက်ပါ Format အတိုင်း ရေးရပါတယ်။

```bibtex
@article{lin2026latex,
  author  = {Lin, Toe Wai},
  title   = {Advanced LaTeX for Software Engineers},
  journal = {Journal of Technical Documentation},
  year    = {2026},
  volume  = {10},
  number  = {2},
  pages   = {45--50}
}
```

* **`@article`**: ဒါက အမျိုးအစားပါ။ စာအုပ်ဆိုရင် `@book`၊ Website ဆိုရင် `@misc` စသဖြင့် ပြောင်းလဲရပါတယ်။
* **`lin2026latex`**: ဒါက **Citation Key** (ID) ဖြစ်ပါတယ်။ `paper.tex` ထဲကနေ လှမ်းခေါ်တဲ့အခါ ဒီ ID ကို သုံးရပါတယ်။
* **Fields**: `author`, `title`, `year` စတဲ့ အချက်အလက်တွေကို `{}` ထဲမှာ ထည့်ရပါတယ်။

---

## ၂။ Data တွေကို အလွယ်တကူ ရှာဖွေထည့်သွင်းနည်း (Google Scholar)
အကို့အနေနဲ့ ဒီ Code တွေကို ကိုယ်တိုင် လက်နဲ့ ရိုက်စရာမလိုပါဘူး။

1.  [Google Scholar](https://scholar.google.com/) ကိုသွားပါ။
2.  ကိုယ်ကိုးကားချင်တဲ့ စာတမ်းအမည်ကို ရိုက်ရှာပါ။
3.  ရလဒ်အောက်က **"Cite"** ခလုတ်ကို နှိပ်ပါ။
4.  ပေါ်လာတဲ့ Box ထဲက **"BibTeX"** ကို နှိပ်ပါ။
5.  ပေါ်လာတဲ့ Code တွေကို Copy ယူပြီး အကို့ရဲ့ `paper.bib` ဖိုင်ထဲမှာ Paste လုပ်လိုက်ရုံပါပဲ။

## ၃။ `paper.tex` ထဲမှာ ပြန်လည်အသုံးပြုပုံ (Citation)
Data တွေ ထည့်ပြီးသွားရင် အကို့ရဲ့ စာတမ်းထဲမှာ အခုလို ပြန်ခေါ်သုံးရပါတယ်။

```latex
% paper.tex ထဲမှာ ရေးရမည့်ပုံစံ
LaTeX is a powerful tool for documentation \cite{lin2026latex}.
```

* အကို Recompile လုပ်လိုက်တဲ့အခါ `\cite{lin2026latex}` နေရာမှာ `[1]` သို့မဟုတ် `(Lin, 2026)` ဆိုပြီး ပေါ်လာပါလိမ့်မယ်။
* စာတမ်းရဲ့ အောက်ဆုံးမှာလည်း **References** ဆိုတဲ့ ခေါင်းစဉ်နဲ့ အချက်အလက်အပြည့်အစုံ အလိုအလျောက် ပေါ်လာမှာ ဖြစ်ပါတယ်။

---

## ၄။ အသုံးများသော အမျိုးအစားများ (Common Types)

| အမျိုးအစား | သုံးရမည့် Tag | လိုအပ်သော အချက်အလက် |
| :--- | :--- | :--- |
| **သုတေသနစာတမ်း** | `@article` | author, title, journal, year |
| **စာအုပ်** | `@book` | author, title, publisher, year |
| **Website** | `@misc` | author, title, howpublished, year, note |
| **Conference** | `@inproceedings` | author, title, booktitle, year |

## ၅။ အရေးကြီးသော အချက် (Special Rules)
* **Author နာမည်များ:** နာမည်နှစ်ခုကြားမှာ `and` ခံပေးရပါတယ်။ ဥပမာ- `author = {Toe Wai Lin and John Doe}`။
* **စာလုံးအကြီး/အသေး:** Title ထဲမှာ စာလုံးအကြီးကို အသေထားချင်ရင် `{ }` နဲ့ အုပ်ပေးရပါတယ်။ ဥပမာ- `title = {Learning {LaTeX} in {Yangon}}`။

အကို့ရဲ့ `paper.bib` ထဲမှာ နမူနာ တစ်ခုလောက် ထည့်စမ်းကြည့်လိုက်ပါ။ ပြီးရင် `paper.tex` ထဲမှာ `\cite{...}` နဲ့ ပြန်ခေါ်ကြည့်ပါ။ အဆင်မပြေတာရှိရင် citation key လေးကို ပြန်စစ်ပေးပါဗျ။

---

အစ်ကိုပြထားတဲ့ `paper.bib` ကုဒ်နမူနာဟာ Academic လောကမှာ အသုံးများတဲ့ **BibTeX Database** ပုံစံဖြစ်ပါတယ်။ ဒီနမူနာထဲက အချက်အလက်တွေကို အခြေခံပြီး တစ်ခုချင်းစီရဲ့ အဓိပ္ပာယ်နဲ့ သတိထားရမယ့် Format စည်းမျဉ်းတွေကို အသေးစိတ် ရှင်းပြပေးပါမယ်။

---

## ၁။ BibTeX Entry တစ်ခု၏ အစိတ်အပိုင်းများ (Explanation)

နမူနာတစ်ခုကို ယူပြီး ကြည့်ရအောင် -

```bibtex
@article{MM26,
    author  = {Adam McCloskey and Pascal Michaillat},
    url     = {https://doi.org/10.1162/rest_a_01456},
    journal = {Review of Economics and Statistics},
    title   = {Critical Values Robust to P-hacking},
    year    = {2026},
}
```

* **`@article`**: ဒါက **Entry Type** ပါ။ မဂ္ဂဇင်း သို့မဟုတ် ဂျာနယ်ထဲက သုတေသနစာတမ်း ဖြစ်ကြောင်း ပြောတာပါ။
* **`MM26`**: ဒါက **Citation Key** (Unique ID) ပါ။ `paper.tex` ထဲမှာ `\cite{MM26}` လို့ ရေးပြီး ခေါ်သုံးရပါမယ်။ (အများအားဖြင့် စာရေးသူရဲ့ နာမည်အစစာလုံးနဲ့ ခုနှစ်ကို တွဲပေးလေ့ရှိပါတယ်)။
* **`author`**: စာရေးသူအမည်များ။
* **`url`**: စာတမ်းကို ဖတ်ရှုနိုင်တဲ့ link (အများအားဖြင့် DOI link တွေကို သုံးပါတယ်)။
* **`journal`**: စာတမ်းကို ဖော်ပြခဲ့တဲ့ ဂျာနယ်အမည်။
* **`title`**: စာတမ်း၏ ခေါင်းစဉ်။
* **`volume` / `number` / `pages`**: စာတမ်းပါရှိတဲ့ ဂျာနယ်ရဲ့ အတွဲအမှတ်၊ စောင်ရေအမှတ်နဲ့ စာမျက်နှာ အညွှန်းများ။

---

## ၂။ Format စည်းမျဉ်းများ (Important Formatting Notes)

BibTeX ရေးတဲ့အခါ အမှားမကင်းအောင် အောက်ပါအချက်တွေကို အထူးသတိပြုရပါမယ်။

### (က) စာရေးသူ အမည်များ (Authors Separation)
စာရေးသူ တစ်ဦးထက်ပိုရင် နာမည်တစ်ခုနဲ့တစ်ခုကြားမှာ **`and`** ဆိုတဲ့ စကားလုံးကိုပဲ သုံးရပါမယ်။ ကော်မာ (`,`) မသုံးရပါဘူး။
* *Correct:* `author = {Adam McCloskey and Pascal Michaillat}`
* *Wrong:* `author = {Adam McCloskey, Pascal Michaillat}`

### (ခ) စာလုံးအကြီး/အသေး ထိန်းသိမ်းခြင်း (Title Case Protection)
LaTeX ဟာ references တွေကို ထုတ်တဲ့အခါ ခေါင်းစဉ်တွေကို အလိုအလျောက် စာလုံးအသေးတွေ ပြောင်းပစ်လေ့ရှိပါတယ်။ အဲဒီလို မဖြစ်စေချင်တဲ့ (အမြဲတမ်း စာလုံးအကြီး ဖြစ်နေစေချင်တဲ့) စာလုံးတွေကို **`{ }`** (Curly braces) နဲ့ အုပ်ပေးရပါတယ်။
* *နမူနာ:* `title = {Resolving {N}ew {K}eynesian Anomalies...}`
* ဒီမှာ `{N}` နဲ့ `{K}` လို့ ရေးထားတဲ့အတွက် PDF မှာ စာလုံးအသေး ပြောင်းမသွားဘဲ အကြီးအတိုင်း ကျန်နေမှာပါ။

### (ဂ) စာမျက်နှာ အကွာအဝေး (Page Ranges)
စာမျက်နှာ နံပါတ်တွေကို ရေးတဲ့အခါ single dash (`-`) ထက်စာရင် double dash (**`--`**) ကို သုံးတာက LaTeX မှာ ပိုပြီး Professional ကျတဲ့ မျဉ်းရှည် (en-dash) ကို ထွက်ပေါ်စေပါတယ်။
* *နမူနာ:* `pages = {1853--1898}`

### (ဃ) ကော်မာ (Commas)
Field တစ်ခုချင်းစီရဲ့ အဆုံးမှာ ကော်မာ (`,`) ပါရပါမယ်။ နောက်ဆုံးစာကြောင်းမှာတော့ ထည့်ရင်လည်းရတယ်၊ မထည့်ရင်လည်း ရပါတယ်။ ဒါပေမဲ့ အမှားကင်းအောင် အမြဲတမ်း ထည့်ထားတာ ပိုကောင်းပါတယ်။

## ၃။ အစ်ကို့ရဲ့ BibTeX List ထဲက ထူးခြားချက်
အစ်ကိုပေးထားတဲ့ list ထဲမှာ `LMS18a` နဲ့ `LMS18b` ဆိုတာကို သတိထားမိမှာပါ။ ဒါက စာရေးသူအုပ်စု တူညီပြီး ထွက်တဲ့နှစ်လည်း တူနေတဲ့အခါ ခွဲခြားရလွယ်အောင် `a` နဲ့ `b` ဆိုပြီး key ထဲမှာ ထည့်ပေးထားတာ ဖြစ်ပါတယ်။ ဒါဟာ Academic writing မှာ standard အလေ့အကျင့်တစ်ခု ဖြစ်ပါတယ်။

---

Website တွေကို ကိုးကားတဲ့အခါမှာ `@misc` ဆိုတဲ့ အမျိုးအစားကို သုံးရပါတယ်။ Website တွေက စာအုပ်တွေလိုမျိုး ပုံသေမဟုတ်ဘဲ ပြောင်းလဲလွယ်တာကြောင့် **ဘယ်နေ့မှာ ကြည့်ရှုခဲ့သလဲ (Access Date)** ဆိုတဲ့ အချက်ကိုပါ ထည့်သွင်းဖို့ လိုအပ်ပါတယ်။

---

### ၁။ `paper.bib` ဖိုင်ထဲမှာ အချက်အလက် ထည့်သွင်းပုံ
Website တစ်ခုအတွက် `paper.bib` ထဲမှာ အခုလို Format အတိုင်း ရေးရပါတယ်။

```bibtex
@misc{overleaf_latex_guide,
  author       = {Overleaf},
  title        = {Learn {LaTeX} in 30 minutes},
  howpublished = {\url{https://www.overleaf.com/learn/latex/Learn_LaTeX_in_30_minutes}},
  year         = {2026},
  note         = {Accessed: 2026-04-19}
}
```

* **`howpublished`**: ဒီနေရာမှာ `\url{...}` ကို သုံးပြီး link ကို ထည့်ရပါတယ်။ (ဒါမှ link က PDF ထဲမှာ နှိပ်လို့ရမှာပါ)။
* **`note`**: ဒီနေရာမှာ `Accessed: [Date]` ဆိုပြီး ကိုယ်ကြည့်ခဲ့တဲ့နေ့စွဲကို ထည့်လေ့ရှိပါတယ်။

---

### ၂။ `paper.tex` ထဲမှာ လိုအပ်တဲ့ ပြင်ဆင်မှု
Link တွေ လှလှပပ ပေါ်လာဖို့အတွက် `paper.tex` ရဲ့ Preamble ထဲမှာ `url` package ပါဝင်နေဖို့ လိုပါတယ်။

```latex
\usepackage{url} % URL link များကို စနစ်တကျ ပြသရန်
```
*(အကယ်၍ `hyperref` ကို သုံးထားပြီးသားဆိုရင်တော့ ဒီ package ကို ထပ်ထည့်စရာ မလိုပါဘူး)*

---

### ၃။ စာသားထဲမှာ ကိုးကားပုံ
တခြားစာတမ်းတွေလိုပဲ `\cite` command ကို သုံးပြီး အခုလို ရေးနိုင်ပါတယ်။

```latex
For beginners, the Overleaf documentation provides an excellent starting point \cite{overleaf_latex_guide}.
```

---

### ၄။ BibTeX Entry ကို အလွယ်ဆုံး ထုတ်ယူနည်း
Website တွေကို ကိုးကားဖို့အတွက် BibTeX code တွေ ထိုင်ရေးနေရတာ လက်ဝင်တယ်ဆိုရင် ဒီ Tool ကို သုံးပါ -

* **ZoteroBib (zbib.org):** 1.  Website ရဲ့ Link ကို Copy ကူးပြီး ဒီ Site ထဲမှာ ထည့်ပါ။
    2.  အောက်က Format နေရာမှာ **BibTeX** ကို ရွေးပါ။
    3.  သူထုတ်ပေးတဲ့ Code ကို Copy ယူပြီး `paper.bib` ထဲမှာ အဆင်သင့် ထည့်သုံးလိုက်ရုံပါပဲ။
  
---

### အနှစ်ချုပ် သတိပြုရန်အချက်များ
1.  **URL Error:** URL ထဲမှာ `_` (underscore) သို့မဟုတ် `%` တွေ ပါနေရင် `\url{...}` ထဲမှာ ထည့်ရေးမှသာ LaTeX က Error မပြမှာ ဖြစ်ပါတယ်။
2.  **Author မသိရင်:** အကယ်၍ ရေးတဲ့သူနာမည် မသိရင် Organization နာမည် (ဥပမာ- `author = {Microsoft}`) ကို သုံးနိုင်ပါတယ်။
3.  **Title:** Website နာမည်တွေမှာ စာလုံးအကြီး/အသေး မပြောင်းစေချင်ရင် `{ }` နဲ့ အုပ်ထားဖို့ မမေ့ပါနဲ့။

---

အစ်ကို့ Project ထဲမှာ ရှိနေတဲ့ **`figures.pdf`** ကတော့ တခြားဖိုင်တွေနဲ့မတူဘဲ နည်းနည်း ထူးခြားပါတယ်။ သူက စာသားတွေရေးထားတဲ့ ဖိုင်မဟုတ်ဘဲ **"Images Assets Container"** (ပုံတွေ သိမ်းထားတဲ့ နေရာ) ဖြစ်ပါတယ်။

Software Engineering concept နဲ့ ပြောရရင် `assets` folder ထဲမှာ ပုံတွေကို စုသိမ်းထားသလိုမျိုးပါပဲ။

---

### ၁။ `figures.pdf` ဆိုတာ ဘာလဲ?
ဒီ template မှာ ပုံတွေကို `.jpg` သို့မဟုတ် `.png` ဖိုင်တွေ တစ်ခုချင်းစီ အများကြီး တင်မယ့်အစား၊ ပုံအားလုံးကို PDF စာမျက်နှာတွေအနေနဲ့ တစ်ဖိုင်တည်းမှာ စုစည်းထားတာ ဖြစ်ပါတယ်။ 
* ဥပမာ - `figures.pdf` ရဲ့ စာမျက်နှာ (၁) မှာ ပထမပုံ၊ စာမျက်နှာ (၂) မှာ ဒုတိယပုံ ဆိုပြီး ရှိနေတာပါ။

### ၂။ ဘာကြောင့် PDF format နဲ့ သိမ်းတာလဲ?
Academic paper တွေမှာ ပုံတွေကို PDF (Vector format) နဲ့ သုံးတာက အားသာချက် ၂ ခု ရှိပါတယ်:
* **High Quality:** PDF ပုံတွေက ဘယ်လောက်ပဲ Zoom ဆွဲဆွဲ ဝါးမသွားပါဘူး (Professional ဆန်ပါတယ်)။
* **Organization:** ပုံတွေ အများကြီးကို ဖိုင်တွေအများကြီးနဲ့ ရှုပ်မနေစေဘဲ တစ်ဖိုင်တည်းနဲ့ စီမံလို့ရပါတယ်။

---

### ၃။ `paper.tex` ထဲမှာ ဒီပုံတွေကို ဘယ်လို လှမ်းခေါ်သုံးမလဲ?
အစ်ကို့ရဲ့ code ထဲမှာ `\newcommand{\pdf}{figures.pdf}` လို့ သတ်မှတ်ထားတာကို မှတ်မိမှာပါ။ အဲဒီပုံတွေကို ထည့်ချင်ရင် အောက်ပါအတိုင်း သုံးရပါတယ် -

```latex
\begin{figure}[h]
    \centering
    % figures.pdf ထဲက စာမျက်နှာ ၁ ကို ပုံအဖြစ် ထည့်မယ်လို့ ပြောတာပါ
    \includegraphics[page=1, width=0.8\textwidth]{\pdf} 
    \caption{This is the first chart from our PDF.}
    \label{fig:chart1}
\end{figure}
```

* **`page=1`**: ဒါက အဓိကကျတဲ့ နေရာပါ။ `figures.pdf` ထဲက ဘယ်နှစ်ခုမြောက် စာမျက်နှာကို ပြမလဲဆိုတာ ရွေးတာပါ။
* **`\pdf`**: ဒါက အပေါ်မှာ သတ်မှတ်ခဲ့တဲ့ `figures.pdf` ဆိုတဲ့ ဖိုင်နာမည်ကို Shortcut ပြန်ခေါ်တာပါ။

---

### ၄။ ကိုယ့်ရဲ့ ကိုယ်ပိုင်ပုံတွေကို ဘယ်လို သုံးမလဲ?
အစ်ကို့မှာ ကိုယ်ပိုင် `.png` သို့မဟုတ် `.jpg` ပုံတွေ ရှိရင် နည်းလမ်း (၂) မျိုး ရှိပါတယ် -

* **နည်းလမ်း (၁):** ပုံတွေကို တစ်ခုချင်းစီ Overleaf ထဲ Upload တင်ပါ။ ပြီးရင် `\includegraphics{your_image.png}` ဆိုပြီး တိုက်ရိုက်ခေါ်သုံးပါ။ (ဒါက အလွယ်ဆုံးနည်းပါ။)
* **နည်းလမ်း (၂):** အစ်ကို့ပုံတွေကို PDF အဖြစ် အရင်ပြောင်း၊ ပြီးရင် `figures.pdf` ထဲမှာ စာမျက်နှာတွေအနေနဲ့ ပေါင်းထည့်ပြီးမှ အပေါ်က code အတိုင်း ပြန်သုံးပါ။

---

### ၅။ `paper.sty` ထဲက Figure Rules နဲ့ ချိတ်ဆက်ပုံ
`paper.sty` ထဲမှာ ပုံတွေနဲ့ ပတ်သက်ပြီး ဒီလို rule လေး ပါပါတယ် -
> `\renewcommand{\figurename}{\textsc{figure}}`

ဒါကြောင့် အစ်ကို ပုံထည့်လိုက်တဲ့အခါ PDF ထဲမှာ **"FIGURE 1"** ဆိုပြီး Small Caps ပုံစံလေးနဲ့ အလိုအလျောက် လှပစွာ ပေါ်လာမှာ ဖြစ်ပါတယ်။

**အနှစ်ချုပ်:**
`figures.pdf` ဆိုတာ အစ်ကို့ရဲ့ စာတမ်းထဲမှာ သုံးမယ့် Graph တွေ၊ Charts တွေကို စုစည်းသိမ်းဆည်းထားတဲ့ **Resource File** တစ်ခုပဲ ဖြစ်ပါတယ်ဗျ။

## ၁။ `paper.bst` (Bibliography Style File)
ဒီဖိုင်က `paper.bib` ထဲက data တွေကို ဘယ်လိုပုံစံနဲ့ PDF ထဲမှာ ဖော်ပြမလဲဆိုတာကို ထိန်းချုပ်တာပါ။

* **သူ့ရဲ့တာဝန်:** အစ်ကိုက စာတမ်းထဲမှာ `\cite{...}` လို့ သုံးလိုက်တဲ့အခါ `[1]` လို့ ပြမလား၊ သို့မဟုတ် `(McCloskey, 2026)` လို့ ပြမလားဆိုတာကို ဒီဖိုင်က ဆုံးဖြတ်ပါတယ်။ References စာရင်းမှာလည်း အမည်ကို အရင်ပြမလား၊ ခုနှစ်ကို အရင်ပြမလားဆိုတာကို သူကပဲ စီမံတာပါ။
* **အလုပ်လုပ်ပုံ:** ဒါကို အစ်ကို ကိုယ်တိုင် ပြင်စရာမလိုပါဘူး။ LaTeX က `paper.tex` ထဲက `\bibliographystyle{paper}` ဆိုတဲ့ စာကြောင်းကို တွေ့တာနဲ့ ဒီ `paper.bst` ကို လှမ်းခေါ်ပြီး အလုပ်လုပ်သွားမှာပါ။

---

## ၂။ `paper.tex` (The Main Document)
ဒါကတော့ အစ်ကို အမြဲတမ်း ဖွင့်ထားရမယ့် **Main Entrance** ဖိုင်ပါ။ စာတမ်းတစ်ခုလုံးရဲ့ Logic တွေ၊ စာသားတွေ အကုန်လုံး ဒီထဲမှာ ရှိပါတယ်။

* **အလုပ်လုပ်ပုံ အဆင့်ဆင့်:**
    1.  **Imports:** ဖိုင်ရဲ့ ထိပ်ဆုံးမှာ `\usepackage{paper}` ဆိုပြီး `paper.sty` ကို လှမ်းခေါ်ပါတယ်။
    2.  **Metadata:** `\title{...}` နဲ့ `\author{...}` တွေကို သတ်မှတ်ပါတယ်။
    3.  **Body:** `\begin{document}` အောက်မှာ တကယ့် စာသားတွေကို Section လိုက် ရေးပါတယ်။
    4.  **Reference Connection:** ဖိုင်ရဲ့ အောက်ဆုံးမှာ `\bibliography{\bib}` လို့ ရေးပြီး `paper.bib` နဲ့ ချိတ်ဆက်လိုက်တာပါ။

---

## ၃။ `paper.pdf` (The Output)
ဒါကတော့ အစ်ကို `Recompile` နှိပ်လိုက်တိုင်း ထွက်လာမယ့် **"Final Product"** ဖြစ်ပါတယ်။ အပေါ်က ဖိုင် ၄ ခုလုံး ပေါင်းစည်းပြီးမှ ဒီ PDF ထွက်လာတာပါ။


### Workflow အကြံပြုချက်
အစ်ကို စာတမ်းရေးတော့မယ်ဆိုရင်:
1.  **`paper.tex`** မှာ စာသားတွေ ရေးပါ။
2.  ကိုးကားစရာရှိရင် **`paper.bib`** ထဲမှာ data သွားထည့်ပါ။
3.  **`paper.tex`** ထဲမှာ `\cite{...}` နဲ့ ပြန်ခေါ်သုံးပါ။
4.  Design ပြင်ချင်မှသာ **`paper.sty`** ထဲကို သွားပါ။

### ၁။ Cross-Referencing (စနစ်တကျ ပြန်လည်ညွှန်းဆိုခြင်း)
LaTeX ရဲ့ အားသာချက်က နံပါတ်တွေကို ကိုယ်တိုင်လိုက်မှတ်စရာ မလိုတာပါ။ `\label` နဲ့ `\ref` ကို ကျွမ်းကျွမ်းကျင်ကျင် သုံးဖို့ အကြံပြုချင်ပါတယ်။
* **Section အတွက်:** `\section{Results}\label{sec:results}`
* **Figure အတွက်:** `\label{fig:architecture}`
* **ခေါ်သုံးပုံ:** `As discussed in Section \ref{sec:results}, the system architecture (see Figure \ref{fig:architecture}) is...`
> **Tip:** အစ်ကိုက စာကြောင်းတွေ အစီအစဉ် ပြောင်းလိုက်ရင်တောင် LaTeX က နံပါတ်တွေကို အလိုအလျောက် ပြန်ညှိပေးသွားမှာပါ။

### ၂။ Mathematical Notation (သင်္ချာနဲ့ Logic များ)
အစ်ကို့ရဲ့ Backend Engineering စာတမ်းတွေမှာ Algorithm တွေ ဒါမှမဟုတ် Performance Logic တွေ ရေးရမယ်ဆိုရင် -
* **Inline Math:** စာကြောင်းထဲမှာ သင်္ကေတထည့်ချင်ရင် `$E=mc^2$` (ဒေါ်လာသင်္ကေတ) ကြားထဲ ထည့်ရေးပါ။
* **Display Math:** ပုံသေနည်းကို အလယ်မှာ သီးသန့်ပြချင်ရင် `\[ ... \]` သို့မဟုတ် `\begin{equation} ... \end{equation}` ကို သုံးပါ။

### ၃။ Version Control (GitHub နဲ့ ချိတ်ဆက်ခြင်း)
အစ်ကိုက Git ကို ကျွမ်းကျင်ပြီးသားဆိုတော့ Overleaf ရဲ့ **GitHub Integration** ကို သုံးဖို့ တိုက်တွန်းပါတယ်။
* Overleaf project ကို GitHub repository နဲ့ ချိတ်ထားနိုင်ပါတယ်။
* ဒါဆိုရင် အစ်ကို့ရဲ့ စာတမ်းက version အလိုက် အမြဲတမ်း backup ရှိနေမှာဖြစ်သလို၊ Offline ရေးချင်ရင်လည်း local စက်ထဲကို `git pull` ဆွဲပြီး VS Code နဲ့ ရေးလို့ရပါတယ်။

### ၄။ Collaborative Writing (Reviewer တွေနဲ့ အလုပ်လုပ်ခြင်း)
အကယ်၍ အစ်ကို့ရဲ့ partner သို့မဟုတ် တခြားသူတွေနဲ့ စာတမ်းကို အတူတူစစ်ချင်ရင် -
* **Comment ပေးခြင်း:** စာသားကို highlight လုပ်ပြီး Overleaf မှာ comment ပေးလို့ရပါတယ်။
* **Track Changes:** ဘယ်သူက ဘာပြင်သွားလဲဆိုတာကို Git history လိုမျိုး ပြန်ကြည့်လို့ရပါတယ်။

### ၅။ Overleaf Shortcuts (မြန်မြန်ဆန်ဆန် အလုပ်လုပ်ရန်)
ဒီ Shortcut လေးတွေက အစ်ကို့အချိန်ကို အများကြီး သက်သာစေမှာပါ -
* **Ctrl + Enter:** Recompile လုပ်ခြင်း (ကုဒ်ပြင်ပြီးတာနဲ့ ချက်ချင်းနှိပ်ကြည့်ပါ)။
* **Ctrl + /:** စာကြောင်းတွေကို Comment ပိတ်ခြင်း/ဖွင့်ခြင်း။
* **Ctrl + F:** စာသားရှာဖွေခြင်း/အစားထိုးခြင်း။
* **Double Click on PDF:** PDF ထဲက စာသားတစ်ခုကို Double click နှိပ်လိုက်ရင် အဲဒီစာသားရှိတဲ့ Code နေရာကို Editor က အလိုအလျောက် ရောက်သွားပါတယ်။
